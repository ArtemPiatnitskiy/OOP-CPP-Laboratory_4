### Почему тут пусто?

Так как мы используем шаблоны, то всю реализацию принято писать в заголовочных файлах
Иногда всё же разбивают на несколько файлов, но это делают для больших проектов, но в нашем случае нет смысла разбивать проект

| Подход                        | Где реализация              | Когда использовать                    |
| ----------------------------- | --------------------------- | ------------------------------------- |
| Всё в `.h`                    | Всё вместе                  | Быстро, просто                        |
| `.h` + `.inl`                 | Разделено, но подключено    | Когда хотим чистый заголовок          |
| `.cpp` + `template class ...` | Только для конкретных типов | Редко, только при узком использовании |
